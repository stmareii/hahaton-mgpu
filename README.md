# Кейс «Анализ соблюдения рекомендаций тестирования детей по истории»

## Задание

Создать инструмент для проверки результатов независимых тестирований, проводимых образовательными организациями, используя данные о проведении тестирования детей на знание истории.

1. Определить нарушения рекомендации о частоте прохождения тестирования (не чаще 1 раза в 3 месяца);
2. Выявить аномалии в данных (технические ошибки, логические противоречия, некорректные или неполные записи);
3. Выявить и интерпретировать паттерны или выбросы, которые могут свидетельствовать о проблемах с качеством данных или нарушении процедуры тестирования.

## Исходные данные

Участникам предоставляется выборка записей о результатах тестирования со следующими полями:
- ФИО ребёнка
- Дата рождения ребёнка
- Дата рождения ребёнка
- ФИО родителя (аналогичные данные)
- Номер результата тестирования
- ОГРН и название школы, направившей на тестирование
- ОГРН и название школы, в которой проводилось тестирование
- Вариант тестирования
- Класс
- Дата тестирования
- Результат тестирования

Исходные данные доступны по [ссылке](https://disk.yandex.ru/d/EzuSBLnCXERTbQ).

## Контекст процедуры

- Тестирование добровольное
- Ребёнок получает направление от школы (где учится или будет учиться) и приходит с ним на площадку тестирования
- Площадка тестирования после проведения вносит сведения в систему учёта
- Рекомендация: проходить тестирование не чаще 1 раза в 3 месяца

## Требования к анализу

1. Проверка соблюдения рекомендации по частоте:

        Представить статистику по нарушениям.

2. Проверка на аномалии.

3. Сценарные выводы:

        Сформулировать рекомендации для организаторов тестирования по улучшению контроля качества
        данных и соблюдения рекомендаций.
        
        Указать, какие типы ошибок чаще всего встречаются и как их можно предотвратить.


## Формат решения

Интерактивный дашборд, ИИ модель.

Презентация с демонстрацией:
- подхода к выявлению нарушений и аномалий
- полученных статистик и примеров
- рекомендаций
